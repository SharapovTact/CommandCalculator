## Git
### R1: Название коммитов должно содержать имя акцентного объекта, с которым были произведены какие либо действия,  например SpecialCounter_create. Список базовых статусов в названии:
- Create object - создали что-то;
- Update object - обновили что-то;
- Update object info - обновили объект и изменили описание его распиновки;
- Delete object - удалениче чего-то.
### R2: Коммит должен содержать 1 акцентный объект, допускается использование/создание побочных объектов, например для специального счётчика был создан D-триггер, который связывает два счётчика.
### R3: Описание коммита включает в себя неформальный вывод по работе и, при необходимости, подробное описание распиновки объекта, который был создан или изменён.
Описание распиновки компонентов выглядит так:
'''D-триггер, имеет два состояния, прозрачное и непрозрачное.
<= D - информационный вход, будет коммутироваться на выход при прозрачном состоянии
<= C - синхроимпульс, на возрастающем фронте переключает состояние триггера между прозрачным и непрозрачным
=> Q - выходной сигнал, возвращает D, при прозрачном состоянии и возвращает лог.0 при непрозрачном состоянии'''
### R4: Коммит в мастер-ветку считается релизом продукта

## VHDL
### R1: Зарезервированные/ключевые слова пишутся с маленьких букв, например begin, end, process.
### R2: Отступ 1 табуляция (в ISE DESIGHN это 3 пробела).
### R3: Типы данных пишутся заглавным регистром.
### R4: Возрастающий/ниспадающий фронт проверяется с помощью (VAR'event and VAR = '1/0').
### R5: Название компонента разделяется '_', каждое слово идёт с заглавной буквы. Название должно отражать назначение компонента.
### R6: При определении типовых компонентов их название должно добавляться в список:
- D_trigger;
- Divider_25MHz - обязательно указание герцовки, при рассчёте нужно отталкиваться от оксиллиратора на 100MHz, который имеется на плате Spartan-6;
### R7: Входы и выходы компонента пишутся с капсом, они должны нести в себе смысл, если было введено какое-то обозначение, то его нужно добавить в этот список и придерживаться его при дальнейшей работе:
- CLK - Тактовый сигнал, согласует 1 операцию;
- RST - Сигнал сброса;
- Q - Выходное значение;
- CLK_XXMHz - Тактовый сигнал с определённой частотой (если это требуется для логики устройства);
